# План: Исправление запуска RPAForge Studio

## Анализ проблемы

### Найденные проблемы

1. **Главная проблема**: Код на строке 42 `main.js` пытается обратиться к `app.isPackaged` на верхнем уровне модуля, когда `electron` API ещё не инициализирован:
   ```javascript
   const isDev = process.env.NODE_ENV === 'development' || !app.isPackaged;
   ```
   Это вызывает ошибку `TypeError: Cannot read properties of undefined (reading 'isPackaged')` при запуске через Node.js вместо Electron.

2. **Проблема с путями**: В последних коммитах были изменения путей:
   - `preload: path.join(__dirname, 'preload.js')` - ожидает preload.js в той же директории
   - `mainWindow.loadFile(path.join(__dirname, '..', 'dist', 'index.html'))` - путь к dist

3. **Конфликтующие проверки isDev**:
   ```typescript
   const isDev = process.env.NODE_ENV === 'development' || !app.isPackaged;
   ```
   и
   ```typescript
   if (isDev && process.env.NODE_ENV === 'development') {
   ```
   Дублирование проверки `process.env.NODE_ENV === 'development'`.

### Структура компиляции

- **tsconfig.node.json**:
  - `rootDir: "."` (packages/studio)
  - `outDir: "dist-electron"`
  - `include: ["electron/main.ts", "electron/python-bridge.ts", ...]`
  - Результат: `electron/main.ts` → `dist-electron/electron/main.js` ✓

- **electron/tsconfig.preload.json**:
  - `outDir: "../dist-electron"`
  - `include: ["preload.ts", ...]`
  - Ожидается: `electron/preload.ts` → `dist-electron/preload.js`
  - Фактически: файлы лежат в `dist-electron/electron/preload.js` ✓

### Рабочие компоненты

- Vite dev server работает (port 5173)
- Файлы успешно компилируются TypeScript
- `dist/index.html` существует и корректен
- Electron установлен (v18.18.2)

## Решение

### 1. Исправить инициализацию isDev в main.ts

**Проблема**: Обращение к `app.isPackaged` на верхнем уровне модуля.

**Решение**: Вынести определение `isDev` внутрь функции `createWindow()` или использовать безопасную проверку:

```typescript
let mainWindow: BrowserWindow | null = null;
let pythonBridge: PythonBridge | null = null;

function createWindow() {
  const isDev = process.env.NODE_ENV === 'development' || !app.isPackaged;
  
  let preloadPath: string;
  
  if (isDev) {
    preloadPath = path.join(process.cwd(), 'dist-electron', 'electron', 'preload.js');
  } else {
    preloadPath = path.join(__dirname, 'preload.js');
  }

  mainWindow = new BrowserWindow({
    width: 1400,
    height: 900,
    minWidth: 1024,
    minHeight: 768,
    webPreferences: {
      preload: preloadPath,
      contextIsolation: true,
      nodeIntegration: false,
    },
    title: 'RPAForge Studio',
  });

  if (isDev) {
    mainWindow.loadURL('http://localhost:5173');
    mainWindow.webContents.openDevTools();
  } else {
    mainWindow.loadFile(path.join(__dirname, '..', 'dist', 'index.html'));
  }

  mainWindow.on('closed', () => {
    mainWindow = null;
  });
}
```

### 2. Упростить логику проверки isDev

**Текущий код**:
```typescript
const isDev = process.env.NODE_ENV === 'development' || !app.isPackaged;
// ...
if (isDev && process.env.NODE_ENV === 'development') {
  mainWindow.loadURL('http://localhost:5173');
```

**Улучшенный код**:
```typescript
const isDev = process.env.NODE_ENV === 'development' || !app.isPackaged;
// ...
if (isDev) {
  mainWindow.loadURL('http://localhost:5173');
```

### 3. Проверить путь к preload.js

В dev режиме используется:
```typescript
preloadPath = path.join(process.cwd(), 'dist-electron', 'electron', 'preload.js');
```

Это правильно, так как `process.cwd()` в dev режиме = `packages/studio`, и файлы компилируются в `dist-electron/electron/`.

В production режиме:
```typescript
preloadPath = path.join(__dirname, 'preload.js');
```

Это также правильно, так как `__dirname` в production = директория `main.js` (т.е. `dist-electron/electron/`), и preload.js будет рядом.

## Альтернативные варианты

### Вариант A: Использовать __dirname вместо process.cwd()

```typescript
if (isDev) {
  preloadPath = path.join(__dirname, 'preload.js');
}
```

**Плюсы**:
- Не зависит от текущей рабочей директории
- Работает как в dev, так и в production

**Минусы**:
- Требует перекомпиляции TypeScript при каждом запуске

### Вариант B: Отложенная инициализация Bridge

Вынести запуск Python Bridge после готовности app:

```typescript
app.whenReady().then(async () => {
  setupIPCHandlers();
  await initializePythonBridge();
  createWindow();
  // ...
});
```

Это уже реализовано, но порядок можно изменить.

## Рекомендуемое решение

**Применить решение 1** - переместить определение `isDev` внутрь `createWindow()` после инициализации Electron.

Это:
1. Исправит ошибку `Cannot read properties of undefined`
2. Сохранит текущую логику работы
3. Не потребует изменений в сборке

## Шаги выполнения

1. Отредактировать `packages/studio/electron/main.ts`:
